\section{Implementation Details}
The application is deployed on a standard LAMP/LEMP stack or locally via XAMPP. The build process uses Vite for compiling the Vue.js frontend assets.

\subsection{Middleware Configuration}
Custom middlewares calculate statistics and manage authentication barriers.

\textbf{Statistics Middleware:} Tracks page views for analytical graphs.
\begin{verbatim}
public function handle(Request $request, Closure $next): Response
{
    $response = $next($request);
    
    if ($request->isMethod('GET') && $response->getStatusCode() === 200) {
        $this->recordStatistics($request);
    }
    
    return $response;
}
\end{verbatim}

\textbf{Advisor Middleware:} Ensures only authenticated advisors can access approval routes.
\begin{verbatim}
public function handle(Request $request, Closure $next): Response
{
    if (!Auth::guard('advisor')->check()) {
        return redirect()->route('advisor_panel.login_page')
            ->with('error', 'Please login to continue');
    }
    return $next($request);
}
\end{verbatim}

\subsection{Frontend Interactions (Inertia.js)}
The frontend handles data sharing via Inertia's `HandleInertiaRequests` middleware, which passes authenticated user data and global settings (like contact info) to every Vue component automatically.

\begin{verbatim}
public function share(Request $request): array
{
    return [
        ...parent::share($request),
        'auth' => [
            'user' => $request->user(),
            'advisor' => Auth::guard('advisor')->user(),
        ],
        'webSettings' => [
            'contact_email' => $contact_email,
            // ...
        ]
    ];
}
\end{verbatim}
