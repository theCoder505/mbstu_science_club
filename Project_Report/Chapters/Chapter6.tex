\section{Unit Testing}
We utilized Laravel's built-in PHPUnit testing suite (now utilizing PestPHP syntax in newer versions).

\subsection{Test Case: Model Instantiation}
\begin{lstlisting}[language=PHP]
public function test_user_can_be_instantiated()
{
    $user = User::factory()->create();
    $this->assertModelExists($user);
}
\end{lstlisting}

\subsection{Test Case: API Endpoint Protection}
\begin{lstlisting}[language=PHP]
public function test_guest_cannot_access_admin_dashboard()
{
    $response = $this->get('/dashboard');
    $response->assertStatus(302); // Redirect
    $response->assertRedirect('/login');
}
\end{lstlisting}

\subsection{Architectural Justification}
The load test focused on \textbf{Read-heavy} operations (Certificate verification, Portfolio browsing) which SQLite handles efficiently. While SQLite has known write-locking limitations, the current concurrent user traffic model for a university club (<5\% concurrent writes) fits within these bounds. However, as noted in the future roadmap, scalability for multi-tenancy will necessitate a migration to MySQL.
